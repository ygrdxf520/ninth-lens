## 图像模型

单位:**元 / 张**(单张输出图像;T2I 与 I2I **同价**,按 `image_count` 计费)

| 模型 | 单价 | T2I | I2I | 说明 |
|------|------|-----|-----|------|
| `qwen-image-2.0` | 0.2 | ✓ | ✓ | ArcReel 默认图像模型,融合 T2I+I2I 同模型 |
| `qwen-image-2.0-pro` | 0.5 | ✓ | ✓ | Pro 版本,文字渲染/真实质感/语义遵循更强 |
| `qwen-image-edit-plus` | 0.2 | — | ✓ | 编辑专用 Plus(多图输出 + 自定义分辨率) |
| `qwen-image-edit-max` | 0.5 | — | ✓ | 编辑专用 Max(工业设计/几何/角色一致性更强) |
| `wan2.7-image` | 0.2 | ✓ | ✓ | 万相 2.7 图像(最高 2K) |
| `wan2.7-image-pro` | 0.5 | ✓ | ✓ | 万相 2.7 Pro(文生图最高 4K;支持交互式编辑/组图/思考模式) |

> **重要**:`qwen-image-2.0` / `qwen-image-2.0-pro` **自身就支持 I2I**(融合模型,T2I + I2I 同 endpoint);只在需要 Max 级角色一致性时才另选 `qwen-image-edit-max/plus`。详见 [千问-文生图.md](./千问-文生图.md)。
>
> ArcReel 不注册经典系列(`qwen-image` / `qwen-image-plus` / `qwen-image-max`)— size 死板/张数固定/异步路径不同,2.0 系列全面替代。

## 视频模型

单位:**元 / 秒**(按输出视频时长计费;wan2.7-r2v 含输入视频时长)

### HappyHorse 系列

统一定价:

| 分辨率 | 单价(元/秒) |
|--------|------------|
| 720P | 0.9 |
| 1080P | 1.6 |

适用模型:
- `happyhorse-1.0-t2v`(文生视频)
- `happyhorse-1.0-i2v`(图生视频)
- `happyhorse-1.0-r2v`(参考生视频)

### 万相 2.7 系列

统一定价:

| 分辨率 | 单价(元/秒) |
|--------|------------|
| 720P | 0.6 |
| 1080P | 1 |

适用模型:
- `wan2.7-t2v`(文生视频)
- `wan2.7-i2v`(图生视频,支持首尾帧 + 续写)
- `wan2.7-r2v`(参考生视频)

> **wan2.7-r2v 计费提示**:`usage.duration` = `input_video_duration` + `output_video_duration`,**含参考视频时长**。前端预估费用时需要把传入的 `reference_video` 时长一并加进去。

## ArcReel 集成口径

在 `lib/config/registry.py::PROVIDER_REGISTRY["dashscope"]` 中:
- 文本:用 `_dashscope_text_pricing` → `PerToken`,`currency="CNY"`
- 图像:用 `_dashscope_image_pricing` → `PerImageFlat`,`currency="CNY"`
- 视频:用 `_dashscope_video_pricing` → `PerSecondMatrix`,`dimensions="resolution_only"`,键 `("720P", None)` / `("1080P", None)`,`currency="CNY"`

视频的 `dimensions="resolution_only"` 而非 `"resolution_audio"`,因为 HappyHorse / Wan2.7 音频**恒开**,价格不随音频开关变化。

## 数据来源

- 文本 / 图像 / 视频价格:百炼控制台 → 模型市场 → 各模型详情页 → 模型价格区(按需切换"千 tokens" / "百万 tokens")
