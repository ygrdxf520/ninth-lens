# 千问图像编辑（Qwen-Image-Edit 系列)

千问图像编辑模型,支持**多图输入和多图输出**,可精确修改图内文字、增删/移动物体、改变主体动作、迁移风格、增强细节。

通用 API 模式详见 [API 概览.md](./API%20概览.md)。

## 模型对照表

| 模型 | 简介 | 张数 | 自定义 size |
|------|------|------|-------------|
| `qwen-image-2.0-pro` 推荐 | 融合模型:T2I + I2I 同模型(详见 [千问-文生图.md](./千问-文生图.md)) | 1-6 | ✓ |
| `qwen-image-2.0` 推荐 | 融合模型加速版 | 1-6 | ✓ |
| `qwen-image-edit-max` | 编辑 Max:工业设计、几何推理、角色一致性更强(snapshot:`qwen-image-edit-max-2026-01-16`) | 1-6 | ✓ |
| `qwen-image-edit-plus` | 编辑 Plus:支持多图输出与自定义分辨率(snapshot:`qwen-image-edit-plus-2025-12-15` / `qwen-image-edit-plus-2025-10-30`) | 1-6 | ✓ |
| `qwen-image-edit` | 基础版:单图编辑和多图融合 | **固定 1** | **不支持** |

**输出规格**:
- 格式:PNG
- 默认分辨率:总像素接近 `1024×1024`,宽高比与输入图(多图时为**最后一张**)相近

**可指定分辨率范围**:
- qwen-image-2.0 系列:总像素 ∈ `[512×512, 2048×2048]`
- qwen-image-edit-max / plus 系列:宽和高均 ∈ `[512, 2048]` px
- qwen-image-edit:**不可指定**(自动按输入图)

**重要**:`size` 系统会以输入值为目标,**将实际宽高调整为最接近的 16 的倍数**。例:传 `1033*1032`,实际输出 `1040*1024`。

## 同步接口(仅同步,无异步)

### Endpoint

```
POST /api/v1/services/aigc/multimodal-generation/generation
```

### 请求体(单图编辑)

```json
{
  "model": "qwen-image-2.0-pro",
  "input": {
    "messages": [
      {
        "role": "user",
        "content": [
          {"image": "https://.../scene.webp"},
          {"text": "在画面右下角...题写一首七言绝句..."}
        ]
      }
    ]
  },
  "parameters": {
    "n": 1,
    "negative_prompt": " ",
    "prompt_extend": true,
    "watermark": false,
    "size": "2048*2048"
  }
}
```

### 请求体(多图融合)

```json
{
  "model": "qwen-image-2.0-pro",
  "input": {
    "messages": [
      {
        "role": "user",
        "content": [
          {"image": "https://.../city.png"},
          {"image": "https://.../cartoon.png"},
          {"text": "使用图一的城市照片作为底图... 三个图二中的卡通形象..."}
        ]
      }
    ]
  },
  "parameters": {
    "n": 1,
    "negative_prompt": " ",
    "prompt_extend": true,
    "watermark": false,
    "size": "2048*2048"
  }
}
```

### `input.messages`（必选)

- **仅支持单轮对话**,数组有且只有一个对象
- `role` 固定 `user`
- `content` 数组:
  - **1 ~ 3 张** `{"image": "..."}` 对象(顺序即图像顺序)
  - **有且只有一个** `{"text": "..."}` 编辑指令

### `image` 输入约束

支持 3 种格式:
- 公网 URL:`https://...`
- OSS 临时 URL:`oss://dashscope-instant/...`
- Base64:`data:{MIME};base64,{data}`

**图像要求**:
- 格式:JPG / JPEG / PNG / BMP / TIFF / WEBP / GIF(仅处理首帧)
- 推荐宽高:`[384, 3072]` px
- 文件大小:≤ 10 MB
- 输出统一为 PNG

### `text`（必选,编辑指令)

- qwen-image-2.0 系列:≤ 1300 token
- 其他模型:≤ 800 token
- 仅支持传**一个** `text` 对象

### `parameters`（可选)

| 参数 | 默认 | 取值 | 说明 |
|------|------|------|------|
| `n` | `1` | `1~6`(除 `qwen-image-edit`,后者固定 1) | 输出张数 |
| `negative_prompt` | — | ≤ 500 字符 | 反向提示词 |
| `size` | 默认见上 | 见模型对照表 | `qwen-image-edit` **不支持**此参数 |
| `prompt_extend` | `true` | bool | 智能改写;`qwen-image-edit` **不支持** |
| `watermark` | `false` | bool | "Qwen-Image" 水印 |
| `seed` | 随机 | `[0, 2147483647]` | 固定提升可复现 |

### 同步响应

```json
{
  "output": {
    "choices": [
      {
        "finish_reason": "stop",
        "message": {
          "role": "assistant",
          "content": [
            {"image": "https://dashscope-result-sz.oss-cn-shenzhen.aliyuncs.com/xxx.png?Expires=..."}
          ]
        }
      }
    ]
  },
  "usage": {
    "height": 2048,
    "image_count": 1,
    "width": 2048
  },
  "request_id": "..."
}
```

`content` 数组长度 = `n`(多图时多个 `{"image": "..."}`)。

## ArcReel 集成要点

### 模型选择策略

千问图像编辑没必要为编辑维护独立模型:**qwen-image-2.0-pro / qwen-image-2.0 已经融合 T2I + I2I**,与文生图同 endpoint。

只在以下场景选编辑专用模型:
- **`qwen-image-edit-max`**:工业设计、几何推理、角色一致性强 — ArcReel 角色一致性确实是痛点,可作 I2I 高质量档
- **`qwen-image-edit-plus`**:与 Max 类似但更便宜
- **`qwen-image-edit`**(基础版):不推荐,固定 1 张 + 不可指定 size + 无 prompt_extend

### 推荐 ModelInfo 注册策略

```python
# T2I + I2I 主力(融合)
"qwen-image-2.0-pro": ModelInfo(
    capabilities=["text_to_image", "image_to_image"],
    pricing=PerImageFlat(rates={"qwen-image-2.0-pro": 0.5}, ...),
),
"qwen-image-2.0": ModelInfo(
    capabilities=["text_to_image", "image_to_image"],
    default=True,
    pricing=PerImageFlat(rates={"qwen-image-2.0": 0.2}, ...),
),

# 角色一致性场景的 I2I 增强
"qwen-image-edit-max": ModelInfo(
    capabilities=["image_to_image"],
    pricing=PerImageFlat(rates={"qwen-image-edit-max": 0.5}, ...),
),
"qwen-image-edit-plus": ModelInfo(
    capabilities=["image_to_image"],
    pricing=PerImageFlat(rates={"qwen-image-edit-plus": 0.2}, ...),
),
# qwen-image-edit(基础版)不进 registry,避免选错
```

### Backend 实现要点

T2I 和 I2I **共用同一 endpoint 与同一请求体结构**,只是 `content` 是否含 `image` 元素的差异:

```python
async def generate(request: ImageGenerationRequest) -> ImageGenerationResult:
    content = []
    if request.reference_images:  # I2I 模式
        for ref in request.reference_images[:3]:  # 上限 3 张
            content.append({"image": _to_url_or_base64(ref)})
    content.append({"text": request.prompt})

    payload = {
        "model": self.model,
        "input": {"messages": [{"role": "user", "content": content}]},
        "parameters": {
            "n": 1,
            "negative_prompt": request.negative_prompt or " ",
            "prompt_extend": False,  # ArcReel 默认关闭,见千问-文生图.md 决策
            "watermark": False,
            "size": request.size or "2048*2048",
        },
    }
    # POST → 解析 output.choices[0].message.content[0].image → 下载到 output_path
```

`capabilities` 校验时:
- 传 `reference_images` 但模型只声明 `text_to_image` → 抛 `ImageCapabilityError`
- 不传 `reference_images` 但模型只声明 `image_to_image` → 同样抛错

### 输入图像上限

ArcReel I2I 当前传 1~N 张参考图,需要在 backend 内**裁剪到 3 张**(千问硬上限),并 log warning。

## 与 wan2.7-image 的差异

| 维度 | qwen-image-2.0 系列 | wan2.7-image 系列 |
|------|--------------------|--------------------|
| Endpoint | `/multimodal-generation/generation`(同步)<br>`/text2image/image-synthesis`(经典系列异步) | `/multimodal-generation/generation`(同步)<br>`/image-generation/generation`(异步) |
| size 系统 | 像素值 `宽*高` | 档位 `1K`/`2K`/`4K` 或像素值 |
| 输入图上限 | 1~3 张 | 0~9 张 |
| 输出张数 | 1~6 | 1~4(关组图)/ 1~12(开组图) |
| 组图功能 | ✗ | ✓(`enable_sequential=true`) |
| 交互式编辑(bbox) | ✗ | ✓(`bbox_list`) |
| 思考模式 | ✗ | ✓(`thinking_mode`) |
| 自定义色板 | ✗ | ✓(`color_palette`) |

详见 [万相2.7-图像.md](./万相2.7-图像.md)。
