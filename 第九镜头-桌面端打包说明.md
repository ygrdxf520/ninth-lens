# 第九镜头 - 桌面端打包说明

## 📦 打包方案

基于 Electron + Python 后端，打包成完整的 Windows 桌面应用安装包。

---

## 🏗️ 项目结构

```
第九镜头/
├── frontend/          # 前端（React + Electron）
│   ├── electron/      # Electron 主进程代码
│   ├── dist/          # 前端构建产物
│   └── ...
├── server/            # 后端（Python FastAPI）
└── python/            # Python 运行环境（打包时放入）
```

---

## 🚀 开发模式运行

### 1. 启动后端
```bash
cd server
pip install -r requirements.txt
python -m uvicorn main:app --host 0.0.0.0 --port 1241
```

### 2. 启动前端开发服务器
```bash
cd frontend
pnpm install
pnpm dev
```

### 3. 启动 Electron
```bash
cd frontend
pnpm electron:dev
```

## 📦 打包成 Windows 安装包

### 前置准备

1. **Node.js** >= 20.19.0
2. **pnpm** >= 10.0.0
3. **Python** 3.10+（用于打包后端）
4. **PyInstaller**（用于打包 Python 后端）

### 步骤一：打包后端（Python）

```bash
# 安装 PyInstaller
pip install pyinstaller

# 进入后端目录
cd server

# 打包后端为单文件 EXE
pyinstaller --onefile --name ninth-lens-server main:app

# 打包产物在 dist/ninth-lens-server.exe
```

### 步骤二：准备 Python 运行环境

方案 A：使用嵌入式 Python（推荐，体积小）
- 下载 Windows embeddable package (64-bit)
- 解压到项目根目录的 python/ 文件夹

方案 B：使用 PyInstaller 打包的单文件
- 直接使用步骤一生成的 EXE 文件

### 步骤三：构建前端

```bash
cd frontend
pnpm install
pnpm build
```

### 步骤四：打包 Electron 应用

```bash
cd frontend

# 打包 Windows 版本
pnpm electron:build:win

# 打包产物在 release/ 目录
# - 第九镜头 Setup 0.18.0.exe（NSIS 安装包）
# - 第九镜头 0.18.0.exe（绿色版）
```

## ⚙️ 配置说明

### 修改后端地址

如果后端不是本地运行，修改 `frontend/electron/main.cjs` 中的配置：

```javascript
const config = {
  backendPort: 1241,
  backendUrl: 'http://localhost:1241'  // 改成你的后端地址
};
```

### 禁用本地后端

如果后端部署在服务器上，不需要本地启动，修改 `startBackend()` 函数，直接返回即可。

## 🎨 自定义配置

### 修改应用图标
1. 准备 .ico 格式的图标文件
2. 放到 `frontend/public/icon.ico`
3. 修改 `package.json` 中的 `build.win.icon` 路径

### 修改安装包信息
编辑 `package.json` 中的 `build` 配置：
- `productName`：产品名称
- `appId`：应用 ID
- `nsis`：安装程序配置

## ⚠️ 注意事项

1. **首次打包**会下载 Electron 二进制文件，比较慢，耐心等待
2. **Windows 打包**建议在 Windows 系统上进行，跨平台打包可能有问题
3. **后端打包**时注意包含所有依赖，特别是 AI 供应商的 SDK
4. **杀毒软件**可能会误报，添加信任即可
5. **体积**：完整打包后大约 200-500MB（包含 Python 环境）

## 🐛 常见问题

### Q: 启动后白屏？
A: 检查前端是否构建成功，dist 目录是否有文件。

### Q: 后端启动失败？
A: 检查端口是否被占用，Python 依赖是否安装完整。

### Q: 打包后找不到后端？
A: 检查 `extraResources` 配置是否正确，后端文件是否被打包进去。

### Q: 如何减小安装包体积？
A: 
- 使用压缩版 Python
- 移除不需要的依赖
- 使用 UPX 压缩 EXE
- 只打包必要的文件
