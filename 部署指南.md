# 第九镜头 - 免费永久部署指南

## 📋 方案说明

- **前端**：部署到 Vercel（完全免费，永久有效，全球 CDN 加速）
- **后端**：部署到 Render 免费版（完全免费，15 分钟无访问会休眠，首次打开需等待 30-60 秒）
- **优点**：一分钱不花，永久可用，品牌为"第九镜头"
- **缺点**：后端免费版有休眠机制，首次访问稍慢

---

## 📝 需要准备的账号

1. **GitHub 账号**（你已经有了 ✓）
2. **Render 账号**：https://render.com/ （用 GitHub 账号登录即可，免费）
3. **Vercel 账号**：https://vercel.com/ （用 GitHub 账号登录即可，免费）

---

## 🚀 第一步：上传代码到 GitHub

### 1.1 创建新仓库

1. 打开 https://github.com/new
2. Repository name 填：`ninth-lens`（或者你喜欢的名字）
3. 选择 **Public**（公开仓库，一键部署需要）
4. 不用勾选 "Add a README file"
5. 点击 **Create repository**

### 1.2 上传代码

**方法一：用 Git 命令行（推荐）**

在本地解压代码包，然后在 ArcReel 目录下执行：

```bash
git init
git add .
git commit -m "初始提交"
git branch -M main
git remote add origin https://github.com/你的用户名/ninth-lens.git
git push -u origin main
```

**方法二：网页上传（适合不熟悉 Git 的）**

1. 在新建的仓库页面，点击 **uploading an existing file**
2. 把 ArcReel 目录里的所有文件和文件夹都拖进去
3. 底部点击 **Commit changes**

---

## 🚀 第二步：部署后端到 Render

### 2.1 一键部署

1. 登录 Render：https://dashboard.render.com/
2. 点击右上角 **New +** → **Web Service**
3. 选择 **Public repository**
4. 输入你的 GitHub 仓库地址：`https://github.com/你的用户名/ninth-lens`
5. 点击 **Continue**

### 2.2 配置服务

- **Name**：`ninth-lens`（或者你喜欢的名字，这会成为网址的一部分）
- **Region**：选一个离你近的（Oregon 或者 Singapore）
- **Branch**：`main`
- **Runtime**：`Docker`
- **Dockerfile Path**：`./Dockerfile`
- **Docker Build Context**：`.`
- **Instance Type**：选 **Free**（免费套餐）

### 2.3 环境变量

点击 **Advanced** → **Add Environment Variable**，添加以下变量：

| Key | Value | 说明 |
|-----|-------|------|
| `AUTH_ENABLED` | `false` | 关闭登录，直接访问 |
| `TZ` | `Asia/Shanghai` | 时区设为中国 |
| `LOG_LEVEL` | `INFO` | 日志级别 |

### 2.4 开始部署

点击 **Create Web Service**

然后等待部署完成，大概需要 5-10 分钟（第一次构建比较慢）。

部署完成后，你会得到一个网址，比如：`https://ninth-lens.onrender.com`

**⚠️ 把这个地址记下来，后面配置前端要用！**

---

## 🚀 第三步：配置前端 API 地址

### 3.1 修改 vercel.json

打开本地代码里的 `frontend/vercel.json` 文件

找到这几行：
```json
"destination": "https://你的后端地址.onrender.com/api/:path*"
```

把 `https://你的后端地址.onrender.com` 改成你刚才在 Render 得到的后端地址。

比如：
```json
"destination": "https://ninth-lens.onrender.com/api/:path*"
```

一共有 3 处需要改（/api、/files、/health）。

### 3.2 提交修改

把修改后的代码提交并推送到 GitHub：

```bash
git add frontend/vercel.json
git commit -m "配置后端 API 地址"
git push
```

---

## 🚀 第四步：部署前端到 Vercel

### 4.1 一键部署

1. 登录 Vercel：https://vercel.com/
2. 点击 **Add New...** → **Project**
3. 选择你刚才的 GitHub 仓库 `ninth-lens`
4. 点击 **Import**

### 4.2 配置项目

- **Project Name**：`ninth-lens`（或者你喜欢的名字）
- **Framework Preset**：选 **Vite**
- **Root Directory**：点击 **Edit**，输入 `frontend`，然后点击 **Continue**

### 4.3 环境变量（可选）

如果你想通过环境变量修改品牌名称，可以在 **Environment Variables** 里添加：

| Key | Value |
|-----|-------|
| `VITE_BRAND_NAME` | `第九镜头` |
| `VITE_BRAND_TAGLINE` | `AI 智能视频创作平台` |

（代码里已经默认配置好了，不添加也可以）

### 4.4 开始部署

点击 **Deploy**

等待 1-2 分钟，部署完成后你会得到一个网址，比如：`https://ninth-lens.vercel.app`

🎉 恭喜！部署完成！

打开这个网址，就能看到你的"第九镜头"AI 视频创作平台了！

## ❓ 常见问题

### Q: Render 免费版休眠是什么意思？
A: 15 分钟没人访问的话，后端服务会暂停。下次有人访问的时候会自动启动，但是第一次打开需要等 30-60 秒。之后就正常了。

### Q: 可以自定义域名吗？
A: 可以！Vercel 和 Render 都支持绑定自己的域名，免费版也可以。

### Q: 数据存在哪里？
A: Render 免费版的文件存储是临时的，重启后数据会丢失。如果需要持久化存储，可以升级 Render 的付费套餐，或者配置外部数据库和对象存储。

### Q: 怎么更新代码？
A: 你把代码推送到 GitHub 的 main 分支，Vercel 和 Render 都会自动重新部署，非常方便。

### Q: 部署失败怎么办？
A: 看一下部署日志里的错误信息，常见原因：
- 代码没上传完整
- 环境变量配置错了
- 端口配置不对（应该是 1241）

## 💡 进阶方案

如果觉得 Render 免费版太慢，可以考虑：

1. **Render 付费版**：$7/月，不会休眠，速度更快
2. **阿里云/腾讯云轻量服务器**：30-50 元/月，用 Docker 一键部署，完全自己掌控
3. **Railway / Fly.io**：类似 Render 的平台，免费额度更多
